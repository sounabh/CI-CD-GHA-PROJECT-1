## **1. BASIC WORKFLOW** 
### File: `.github/workflows/1-basic.yml`

```yaml
# The name that appears in the GitHub Actions UI
name: 1. Basic Workflow

# TRIGGER: When does this workflow run?
# This runs when you push code to the main branch
on:
  push:
    branches:
      - main

# JOBS: What tasks should run?
jobs:
  # Job ID (you reference this in other jobs)
  build:
    # Human-readable name
    name: Build and Test
    
    # Which OS to run on (GitHub provides these virtual machines)
    runs-on: ubuntu-latest
    
    # STEPS: Individual commands that run in sequence
    steps:
      # Step 1: Download your repository code
      # 'uses' means we're using a pre-built action from GitHub marketplace
      - name: Checkout code
        uses: actions/checkout@v4
      
      # Step 2: Install Node.js
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      
      # Step 3: Install dependencies
      # 'run' means execute a shell command
      - name: Install dependencies
        run: npm install
      
      # Step 4: Run tests
      - name: Run tests
        run: npm test
      
      # Step 5: Build the project
      - name: Build
        run: npm run build
```

**📚 CONCEPTS EXPLAINED:**
- **name**: Identifies your workflow
- **on**: Event triggers (push, pull_request, schedule, etc.)
- **jobs**: Collection of steps that run on the same runner
- **runs-on**: The operating system (ubuntu-latest, windows-latest, macos-latest)
- **steps**: Sequential tasks
- **uses**: Use pre-built actions
- **run**: Execute shell commands

## **2. CONTEXTS**
### File: `.github/workflows/2-contexts.yml`

```yaml
name: 2. Contexts

on: push

jobs:
  show-contexts:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      
      # GITHUB CONTEXT: Information about the workflow run
      - name: Show GitHub Context
        run: |
          echo "Repository: ${{ github.repository }}"
          echo "Event Name: ${{ github.event_name }}"
          echo "SHA: ${{ github.sha }}"
          echo "Actor: ${{ github.actor }}"
          echo "Ref: ${{ github.ref }}"
          echo "Workflow: ${{ github.workflow }}"
      
      # RUNNER CONTEXT: Information about the runner executing the job
      - name: Show Runner Context
        run: |
          echo "OS: ${{ runner.os }}"
          echo "Temp Directory: ${{ runner.temp }}"
          echo "Workspace: ${{ runner.workspace }}"
      
      # JOB CONTEXT: Information about the current job
      - name: Show Job Context
        run: |
          echo "Job Status: ${{ job.status }}"
      
      # ENV CONTEXT: Environment variables
      - name: Show Environment Variables
        env:
          CUSTOM_VAR: "Hello from CI/CD"
        run: |
          echo "Custom Variable: ${{ env.CUSTOM_VAR }}"
          echo "PATH: ${{ env.PATH }}"
```

**📚 CONTEXTS EXPLAINED:**
- **`${{ }}`**: Expression syntax to access context data
- **github**: Info about the repo, event, user
- **runner**: Info about the machine running the job
- **job**: Info about the current job
- **env**: Environment variables
- **secrets**: Encrypted secrets (we'll cover later)

## **3. MULTIPLE JOBS**
### File: `.github/workflows/3-multiple-jobs.yml`

```yaml
name: 3. Multiple Jobs

on: push

jobs:
  # First job: Install and test
  test:
    name: Run Tests
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm install
      - run: npm test
  
  # Second job: Build (runs in parallel by default)
  build:
    name: Build Application
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm install
      - run: npm run build
  
  # Third job: Deploy (waits for test and build)
  deploy:
    name: Deploy to Staging
    runs-on: ubuntu-latest
    # NEEDS: This job only runs after 'test' and 'build' succeed
    needs: [test, build]
    steps:
      - name: Deploy
        run: echo "Deploying application..."
```

**📚 MULTIPLE JOBS EXPLAINED:**
- Jobs run in **parallel** by default
- **needs**: Creates dependencies (sequential execution)
- Each job runs on a **fresh runner** (isolated environment)
- Jobs can run on **different operating systems**

## **4. MATRIX STRATEGY**
### File: `.github/workflows/4-matrix.yml`

```yaml
name: 4. Matrix Strategy

on: push

jobs:
  test-matrix:
    name: Test on Node ${{ matrix.node }} and ${{ matrix.os }}
    runs-on: ${{ matrix.os }}
    
    # MATRIX: Test multiple configurations
    strategy:
      # Don't cancel all jobs if one fails
      fail-fast: false
      matrix:
        # Test on multiple Node versions
        node: [18, 20, 21]
        # Test on multiple operating systems
        os: [ubuntu-latest, windows-latest, macos-latest]
        # This creates 9 jobs (3 nodes × 3 OS)
        
        # You can also include/exclude specific combinations
        include:
          # Add an extra job with experimental Node version
          - node: 22
            os: ubuntu-latest
            experimental: true
        
        exclude:
          # Don't test Node 18 on macOS (just as example)
          - node: 18
            os: macos-latest
    
    # Mark experimental jobs as allowed to fail
    continue-on-error: ${{ matrix.experimental == true }}
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js ${{ matrix.node }}
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
      
      - run: npm install
      - run: npm test
```

**📚 MATRIX EXPLAINED:**
- **matrix**: Run the same job with different configurations
- **fail-fast**: Control whether to stop all jobs on first failure
- **include**: Add extra combinations
- **exclude**: Remove specific combinations
- **continue-on-error**: Allow certain jobs to fail

## **5. CACHING**
### File: `.github/workflows/5-cache.yml`

```yaml
name: 5. Caching Dependencies

on: push

jobs:
  build-with-cache:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          # BUILT-IN CACHING: Cache npm dependencies automatically
          cache: 'npm'
      
      - name: Install dependencies
        run: npm install
      
      # MANUAL CACHING: Cache custom directories
      - name: Cache build output
        uses: actions/cache@v3
        with:
          # What to cache
          path: |
            dist/
            build/
            .next/cache
          # Cache key (if package.json changes, cache is invalidated)
          key: ${{ runner.os }}-build-${{ hashFiles('**/package-lock.json') }}
          # Fallback keys if exact match not found
          restore-keys: |
            ${{ runner.os }}-build-
            ${{ runner.os }}-
      
      - name: Build
        run: npm run build
      
      - name: Show cache info
        run: |
          echo "Cache hit: ${{ steps.cache.outputs.cache-hit }}"
```

**📚 CACHING EXPLAINED:**
- **Why cache?**: Speed up workflows by reusing dependencies
- **cache (setup-node)**: Automatic caching for npm/yarn/pnpm
- **actions/cache**: Manual caching for any files/folders
- **key**: Unique identifier (use file hashes for smart invalidation)
- **restore-keys**: Fallback if exact key doesn't match
- **hashFiles()**: Creates hash from file contents

## **6. ARTIFACTS**
### File: `.github/workflows/6-artifacts.yml`

```yaml
name: 6. Artifacts

on: push

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm install
      - run: npm run build
      
      # UPLOAD ARTIFACTS: Save files from this job
      - name: Upload build artifacts
        uses: actions/upload-artifact@v4
        with:
          # Name for the artifact
          name: build-output
          # What files to upload
          path: |
            dist/
            build/
          # How long to keep (default: 90 days)
          retention-days: 30
      
      # Generate test coverage report
      - run: npm run test:coverage
      
      - name: Upload coverage report
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: coverage/
          retention-days: 7
  
  # Separate job that uses the artifacts
  deploy:
    runs-on: ubuntu-latest
    needs: build
    steps:
      # DOWNLOAD ARTIFACTS: Retrieve files from previous job
      - name: Download build artifacts
        uses: actions/download-artifact@v4
        with:
          name: build-output
          path: ./dist
      
      - name: List downloaded files
        run: ls -la ./dist
      
      - name: Deploy
        run: echo "Deploying files from ./dist"
```

**📚 ARTIFACTS EXPLAINED:**
- **Purpose**: Share files between jobs or download after workflow completes
- **upload-artifact**: Save files (build outputs, reports, logs)
- **download-artifact**: Retrieve files in another job
- **retention-days**: How long GitHub stores the files
- **Use cases**: Build outputs, test reports, logs, screenshots

## **7. OUTPUTS**
### File: `.github/workflows/7-outputs.yml`

```yaml
name: 7. Job Outputs

on: push

jobs:
  build-info:
    runs-on: ubuntu-latest
    # DEFINE OUTPUTS: Data this job will share
    outputs:
      version: ${{ steps.get-version.outputs.version }}
      build-time: ${{ steps.get-time.outputs.time }}
      should-deploy: ${{ steps.check-deploy.outputs.deploy }}
    
    steps:
      - uses: actions/checkout@v4
      
      # Step that creates an output
      - name: Get version from package.json
        id: get-version
        run: |
          VERSION=$(node -p "require('./package.json').version")
          echo "version=$VERSION" >> $GITHUB_OUTPUT
          echo "Extracted version: $VERSION"
      
      - name: Get build time
        id: get-time
        run: |
          TIME=$(date +'%Y-%m-%d %H:%M:%S')
          echo "time=$TIME" >> $GITHUB_OUTPUT
      
      - name: Check if should deploy
        id: check-deploy
        run: |
          if [[ "${{ github.ref }}" == "refs/heads/main" ]]; then
            echo "deploy=true" >> $GITHUB_OUTPUT
          else
            echo "deploy=false" >> $GITHUB_OUTPUT
          fi
  
  # Job that uses the outputs
  use-outputs:
    runs-on: ubuntu-latest
    needs: build-info
    steps:
      - name: Display outputs
        run: |
          echo "Version: ${{ needs.build-info.outputs.version }}"
          echo "Build Time: ${{ needs.build-info.outputs.build-time }}"
          echo "Should Deploy: ${{ needs.build-info.outputs.should-deploy }}"
  
  deploy:
    runs-on: ubuntu-latest
    needs: build-info
    # Only run if should-deploy is true
    if: needs.build-info.outputs.should-deploy == 'true'
    steps:
      - name: Deploy version ${{ needs.build-info.outputs.version }}
        run: echo "Deploying..."
```

**📚 OUTPUTS EXPLAINED:**
- **outputs**: Share data between jobs
- **steps.STEP_ID.outputs.NAME**: Reference step outputs
- **$GITHUB_OUTPUT**: Write outputs in steps
- **needs.JOB_ID.outputs.NAME**: Access outputs from other jobs
- **Syntax**: `echo "key=value" >> $GITHUB_OUTPUT`

## **8. CONDITIONS**
### File: `.github/workflows/8-conditions.yml`

```yaml
name: 8. Conditions

on:
  push:
    branches: [main, develop]
  pull_request:

jobs:
  conditional-job:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      # STEP-LEVEL CONDITION: Only run on main branch
      - name: Production deployment
        if: github.ref == 'refs/heads/main'
        run: echo "Deploying to production"
      
      # Run only on develop branch
      - name: Staging deployment
        if: github.ref == 'refs/heads/develop'
        run: echo "Deploying to staging"
      
      # Run only on pull requests
      - name: PR check
        if: github.event_name == 'pull_request'
        run: echo "Running PR checks"
      
      # Run only for specific actor
      - name: Admin only
        if: github.actor == 'your-username'
        run: echo "Running admin task"
      
      # Complex condition with AND/OR
      - name: Complex condition
        if: |
          github.event_name == 'push' &&
          (github.ref == 'refs/heads/main' || github.ref == 'refs/heads/develop')
        run: echo "Push to main or develop"
      
      # Condition with always() - runs even if previous steps failed
      - name: Cleanup
        if: always()
        run: echo "This always runs"
      
      # Run only if previous steps succeeded
      - name: Success only
        if: success()
        run: echo "All previous steps succeeded"
      
      # Run only if previous steps failed
      - name: On failure
        if: failure()
        run: echo "Something failed"
  
  # JOB-LEVEL CONDITION
  production-deploy:
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    steps:
      - name: Deploy
        run: echo "This job only runs on push to main"
```

**📚 CONDITIONS EXPLAINED:**
- **if**: Control when jobs/steps run
- **Common conditions**:
  - `github.ref == 'refs/heads/main'` - specific branch
  - `github.event_name == 'push'` - event type
  - `github.actor == 'username'` - who triggered
- **Status functions**:
  - `success()` - all previous steps succeeded
  - `failure()` - any previous step failed
  - `always()` - run regardless of status
  - `cancelled()` - workflow was cancelled
- **Operators**: `&&` (AND), `||` (OR), `!` (NOT)

```yaml
      - name: Deploy to cloud
        uses: some-deploy-action@v1
        with:
          api-token: ${{ secrets.DEPLOY_TOKEN }}
          
      # GITHUB_TOKEN: Automatically provided secret
      - name: Use GitHub Token

```

**📚 SECRETS EXPLAINED:**

**How to add secrets in GitHub:**
1. Go to: Repository → Settings → Secrets and variables → Actions
2. Click "New repository secret"
3. Add: `API_KEY`, `DATABASE_URL`, etc.

**Types of secrets:**
- **Repository secrets**: Available to all workflows in the repo
- **Environment secrets**: Specific to an environment
- **Organization secrets**: Shared across multiple repos

**Best practices:**
- Never hardcode secrets in workflow files
- Secrets are automatically masked in logs
- `GITHUB_TOKEN` is automatically created for each workflow

## **11. REUSABLE WORKFLOWS**
### File: `.github/workflows/reusable-deploy.yml`

```yaml
name: Reusable Deploy Workflow

# WORKFLOW_CALL: Makes this workflow reusable
on:
  workflow_call:
    # INPUTS: Parameters passed to this workflow
    inputs:
      environment:
        required: true
        type: string
        description: 'Deployment environment'
      node-version:
        required: false
        type: string
        default: '20'
        description: 'Node.js version'
    
    # SECRETS: Secrets passed from calling workflow
    secrets:
      deploy-token:
        required: true
        description: 'Deployment token'
    
    # OUTPUTS: Data this workflow returns
    outputs:
      deployment-url:
        description: 'URL of the deployment'
        value: ${{ jobs.deploy.outputs.url }}

jobs:
  deploy:
    runs-on: ubuntu-latest
    outputs:
      url: ${{ steps.deploy.outputs.url }}
    
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ inputs.node-version }}
      
      - run: npm install
      - run: npm run build
      
      - name: Deploy to ${{ inputs.environment }}
        id: deploy
        env:
          DEPLOY_TOKEN: ${{ secrets.deploy-token }}
        run: |
          echo "Deploying to ${{ inputs.environment }}..."
          URL="https://${{ inputs.environment }}.example.com"
          echo "url=$URL" >> $GITHUB_OUTPUT
          echo "Deployed to: $URL"
```


- **workflow_call**: Trigger that makes workflow reusable
- **inputs**: Parameters with types and defaults
- **secrets**: Encrypted values passed to reusable workflow
- **outputs**: Return values from reusable workflow
- **uses**: Call a reusable workflow (local or from another repo)
- **Benefits**: DRY principle, consistency, easier maintenance

## **12. CONCURRENCY**
### File: `.github/workflows/12-concurrency.yml`

```yaml
name: 12. Concurrency Control

on:
  push:
    branches: [main, develop]
  pull_request:

# WORKFLOW-LEVEL CONCURRENCY
concurrency:
  # Group name: only one workflow in this group runs at a time
  group: ${{ github.workflow }}-${{ github.ref }}
  # Cancel in-progress runs when new one starts
  cancel-in-progress: true

jobs:
  deploy-staging:
    runs-on: ubuntu-latest
    # JOB-LEVEL CONCURRENCY
    concurrency:
      group: deploy-staging
      cancel-in-progress: false  # Don't cancel deployments
    steps:
      - uses: actions/checkout@v4
      - name: Deploy
        run: |
          echo "Deploying to staging..."
          sleep 30
  
  deploy-production:
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    concurrency:
      # Unique group per environment
      group: deploy-production-${{ github.ref }}
      cancel-in-progress: false
    steps:
      - uses: actions/checkout@v4
      - name: Deploy
        run: |
          echo "Deploying to production..."
          sleep 30
  
  test:
    runs-on: ubuntu-latest
    # Allow multiple test runs concurrently
    # No concurrency control for this job
    steps:
      - uses: actions/checkout@v4
      - name: Run tests
        run: npm test
```

**📚 CONCURRENCY EXPLAINED:**
- **Purpose**: Control how many workflow runs execute simultaneously
- **group**: Identifier for the concurrency group
- **cancel-in-progress**: 
  - `true` - Cancel old runs when new one starts (good for CI)
  - `false` - Queue new runs (good for deployments)
- **Use cases**:
  - Prevent multiple deployments
  - Save CI minutes by cancelling outdated PRs
  - Ensure only one release at a time

```yaml
name: 13. Deployment Environments

```

**📚 ENVIRONMENTS EXPLAINED:**

**Setting up in GitHub:**
1. Repository → Settings → Environments
2. Create environments: `staging`, `production`

**📚 SCHEDULED WORKFLOWS EXPLAINED:**

**Cron syntax:** `minute hour day month weekday`
```
┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of week (0 - 6) (Sunday to Saturday)
│ │ │ │ │
* * * * *
```

**Examples:**
- `'0 0 * * *'` - Daily at midnight
- `'*/15 * * * *'` - Every 15 minutes
- `'0 9 * * 1-5'` - Weekdays at 9 AM
- `'0 0 1 * *'` - First day of every month

**Use cases:**
- Nightly builds and tests
- Dependency updates
- Database backups
- Health checks
- Cleanup tasks
- Report generation

**Important notes:**
- Uses UTC timezone
- Minimum interval: 5 minutes
- Scheduled workflows may be delayed during peak times
- Disabled if no repository activity for 60 days

## **16. FINAL COMPLETE DEPLOYMENT WORKFLOW**
### File: `.github/workflows/16-final-deployment.yml`

```yaml
name: 16. Complete CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]
  workflow_dispatch:
    inputs:
      deploy-env:
        description: 'Force deployment to environment'
        type: choice
        options:
          - none
          - staging
          - production
        default: 'none'

# Prevent concurrent deployments
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: ${{ github.event_name == 'pull_request' }}

env:
  NODE_VERSION: '20'
  CACHE_KEY_PREFIX: 'node-modules'

jobs:
  # ============================================
  # JOB 1: CODE QUALITY & LINTING
  # ============================================
  code-quality:
    name: Code Quality Checks
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for better analysis
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run linter
        run: npm run lint
      
      - name: Check code formatting
        run: npm run format:check
      
      - name: Type checking
        run: npm run type-check
        continue-on-error: true  # Don't fail build, just warn

  # ============================================
  # JOB 2: SECURITY AUDIT
  # ============================================
  security-audit:
    name: Security Audit
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
      
      - name: Run security audit
        run: npm audit --audit-level=high
        continue-on-error: true
      
      - name: Check for vulnerable dependencies
        run: |
          npm install -g npm-check-updates
          ncu --doctor --doctorTest "npm audit"

  # ============================================
  # JOB 3: TEST MATRIX (Multiple Node versions)
  # ============================================
  test:
    name: Test (Node ${{ matrix.node }})
    runs-on: ${{ matrix.os }}
    
    strategy:
      fail-fast: false
      matrix:
        node: [18, 20]
        os: [ubuntu-latest]
        include:
          - node: 20
            os: windows-latest
          - node: 20
            os: macos-latest
    
    # Service containers for integration tests
    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_PASSWORD: postgres
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
        ports:
          - 5432:5432
      
      redis:
        image: redis:7
        options: >-
          --health-cmd "redis-cli ping"
        ports:
          - 6379:6379
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js ${{ matrix.node }}
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run unit tests
        run: npm run test:unit
      
      - name: Run integration tests
        env:
          DATABASE_URL: postgresql://postgres:postgres@localhost:5432/test
          REDIS_URL: redis://localhost:6379
        run: npm run test:integration
      
      - name: Generate coverage report
        run: npm run test:coverage
      
      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v3
        with:
          token: ${{ secrets.CODECOV_TOKEN }}
          files: ./coverage/coverage-final.json
          flags: node-${{ matrix.node }}
      
      - name: Upload test results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: test-results-${{ matrix.os }}-node-${{ matrix.node }}
          path: |
            coverage/
            test-results/
          retention-days: 7

  # ============================================
  # JOB 4: BUILD
  # ============================================
  build:
    name: Build Application
    runs-on: ubuntu-latest
    needs: [code-quality, test]
    
    outputs:
      version: ${{ steps.package-version.outputs.version }}
      build-time: ${{ steps.build-info.outputs.time }}
    
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Get package version
        id: package-version
        run: |
          VERSION=$(node -p "require('./package.json').version")
          echo "version=$VERSION" >> $GITHUB_OUTPUT
          echo "Building version: $VERSION"
      
      - name: Build application
        run: npm run build
        env:
          NODE_ENV: production
      
      - name: Record build info
        id: build-info
        run: |
          echo "time=$(date -u +'%Y-%m-%dT%H:%M:%SZ')" >> $GITHUB_OUTPUT
          echo "sha=${{ github.sha }}" >> build-info.txt
          echo "ref=${{ github.ref }}" >> build-info.txt
          echo "version=${{ steps.package-version.outputs.version }}" >> build-info.txt
      
      - name: Upload build artifacts
        uses: actions/upload-artifact@v4
        with:
          name: build-artifacts
          path: |
            dist/
            build/
            build-info.txt
          retention-days: 30
      
      - name: Cache build output
        uses: actions/cache@v3
        with:
          path: dist/
          key: build-${{ github.sha }}

  # ============================================
  # JOB 5: DOCKER BUILD (Optional)
  # ============================================
  docker-build:
    name: Build Docker Image
    runs-on: ubuntu-latest
    needs: build
    if: github.event_name != 'pull_request'
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Download build artifacts
        uses: actions/download-artifact@v4
        with:
          name: build-artifacts
          path: ./dist
      
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3
      
      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}
      
      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: |
            ${{ secrets.DOCKER_USERNAME }}/myapp:latest
            ${{ secrets.DOCKER_USERNAME }}/myapp:${{ needs.build.outputs.version }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  # ============================================
  # JOB 6: DEPLOY TO STAGING
  # ============================================
  deploy-staging:
    name: Deploy to Staging
    runs-on: ubuntu-latest
    needs: [build, security-audit]
    if: |
      github.ref == 'refs/heads/develop' ||
      inputs.deploy-env == 'staging'
    
    environment:
      name: staging
      url: https://staging.example.com
    
    concurrency:
      group: deploy-staging
      cancel-in-progress: false
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Download build artifacts
        uses: actions/download-artifact@v4
        with:
          name: build-artifacts
          path: ./dist
      
      - name: Deploy to staging
        env:
          DEPLOY_TOKEN: ${{ secrets.STAGING_DEPLOY_TOKEN }}
          API_URL: ${{ secrets.STAGING_API_URL }}
        run: |
          echo "Deploying version ${{ needs.build.outputs.version }} to staging"
          # Your deployment script here
          # ./scripts/deploy.sh staging
      
      - name: Run smoke tests
        run: |
          echo "Running smoke tests against staging"
          # npm run test:smoke -- --url https://staging.example.com
      
      - name: Notify deployment
        if: always()
        run: |
          echo "Deployment status: ${{ job.status }}"
          # Send Slack/Discord notification

  # ============================================
  # JOB 7: DEPLOY TO PRODUCTION
  # ============================================
  deploy-production:
    name: Deploy to Production
    runs-on: ubuntu-latest
    needs: [build, docker-build]
    if: |
      github.ref == 'refs/heads/main' ||
      inputs.deploy-env == 'production'
    
    environment:
      name: production
      url: https://example.com
    
    concurrency:
      group: deploy-production
      cancel-in-progress: false
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Download build artifacts
        uses: actions/download-artifact@v4
        with:
          name: build-artifacts
          path: ./dist
      
      - name: Create deployment
        uses: actions/github-script@v7
        with:
          script: |
            const deployment = await github.rest.repos.createDeployment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              ref: context.sha,
              environment: 'production',
              required_contexts: [],
              auto_merge: false
            });
            console.log('Deployment created:', deployment.data.id);
      
      - name: Deploy to production
        env:
          DEPLOY_TOKEN: ${{ secrets.PRODUCTION_DEPLOY_TOKEN }}
          API_URL: ${{ secrets.PRODUCTION_API_URL }}
        run: |
          echo "Deploying version ${{ needs.build.outputs.version }} to production"
          echo "Build time: ${{ needs.build.outputs.build-time }}"
          # ./scripts/deploy.sh production
      
      - name: Health check
        run: |
          sleep 10
          curl -f https://example.com/health || exit 1
      
      - name: Update deployment status
        if: always()
        uses: actions/github-script@v7
        with:
          script: |
            const state = '${{ job.status }}' === 'success' ? 'success' : 'failure';
            // Update deployment status
            console.log('Deployment', state);

  # ============================================
  # JOB 8: POST-DEPLOYMENT VERIFICATION
  # ============================================
  verify-deployment:
    name: Verify Deployment
    runs-on: ubuntu-latest
    needs: deploy-production
    if: success()
    
    steps:
      - name: Run E2E tests
        run: |
          echo "Running end-to-end tests"
          # npm run test:e2e -- --url https://example.com
      
      - name: Performance tests
        run: |
          echo "Running performance tests"
          # npm run test:performance
      
      - name: Create GitHub Release
        if: github.ref == 'refs/heads/main'
        uses: actions/create-release@v1
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          tag_name: v${{ needs.build.outputs.version }}
          release_name: Release v${{ needs.build.outputs.version }}
          body: |
            Deployed at ${{ needs.build.outputs.build-time }}
            SHA: ${{ github.sha }}
          draft: false
          prerelease: false

  # ============================================
  # JOB 9: CLEANUP
  # ============================================
  cleanup:
    name: Cleanup
    runs-on: ubuntu-latest
    needs: [deploy-staging, deploy-production]
    if: always()
    
    steps:
      - name: Clean old artifacts
        uses: actions/github-script@v7
        with:
          script: |
            const days = 7;
            const timestamp = new Date();
            timestamp.setDate(timestamp.getDate() - days);
            
            const artifacts = await github.rest.actions.listArtifactsForRepo({
              owner: context.repo.owner,
              repo: context.repo.repo,
              per_page: 100
            });
            
            for (const artifact of artifacts.data.artifacts) {
              if (new Date(artifact.created_at) < timestamp) {
                console.log(`Deleting artifact: ${artifact.name}`);
                await github.rest.actions.deleteArtifact({
                  owner: context.repo.owner,
                  repo: context.repo.repo,
                  artifact_id: artifact.id
                });
              }
            }
```
